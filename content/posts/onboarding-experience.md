Most onboarding flows are tutorials about a product. The good ones are tutorials _using_ a product.

Resend gets this right. You land on the dashboard and the first screen is "Send your first email." Not "watch this video." Not "let's set up your workspace." You get an API key, a code snippet in your language, and a button that actually sends the email. Three steps, real output, done.

NotebookLM does the same thing. You arrive, you drop in a source, and within a minute the AI is answering questions about it. You're not configuring anything. You're not reading docs. You're already using the tool for the thing you came to do.

The pattern is the same in both cases:

1. Defer friction. Don't ask for preferences, integrations, or team names on day one. Ask when it matters.
2. Give real output immediately. A sent email. A generated question. Not a demo, not a simulation. Something the user actually made.
3. One path forward. No three-column layout of features. One obvious next step.

The places where onboarding fails usually do the opposite. They front-load configuration, show you everything at once, and call it "powerful."

Products that let you succeed in five minutes don't need to explain how good they are.
